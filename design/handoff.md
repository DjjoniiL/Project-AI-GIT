# Handoff Spec: Конструктор заказов одежды с принтом — экран конструктора

Итоговый спек для разработки (Этап 2 из плана, Итерация 3 из `specification.md` раздел 2.1). Основан на финальном согласованном макете [`maket-4.html`](maket-4.html) (согласован 2026-07-01, после `design-critique`, `accessibility-review`, `design-system`, `ux-copy`).

## Overview

Единственный экран приложения — вкладка внутри карточки сделки Битрикс24 (`CRM_DEAL_DETAIL_TAB`). Пользователь — менеджер печатного цеха, работает на десктопе внутри CRM. Экран собирает все параметры заказа (изделие, ткань, цвет, размеры, принт, макет, комментарий) и по кнопке отправляет их в сделку.

## Layout

- Референс-ширина в макете — 680px (ограничение самого инструмента предпросмотра, не продуктовое требование).
- CSS Grid, 2 равные колонки (`1fr 1fr`), `gap: 16px`, `align-items: start`.
- Левая колонка — `flex-direction: column`, `gap: 14px`, содержит все поля формы.
- Правая колонка — `position: sticky; top: 0`, `gap: 10px`, содержит превью + селекторы изделия + кнопку экспорта.
- **Открытый вопрос — брейкпоинты**: макет ни разу не проверялся на другой ширине, кроме 680px. Реальная ширина вкладки Битрикс24 неизвестна и может быть у́же (боковая панель) или шире (полноэкранная вкладка). Нужно решение до старта вёрстки: либо зафиксировать минимальную ширину с горизонтальным скроллом, либо спроектировать брейкпоинт, при котором колонки складываются в одну.

## Design Tokens

Цвета сейчас — семантические CSS-переменные CDS (хост-дизайн-система инструмента предпросмотра), не Ant-токены. При переносе — смаппить на `ConfigProvider` theme:

| Токен CDS | Роль | Ant-эквивалент (предложение) |
|---|---|---|
| `--surface-2` | Фон карточек формы | `colorBgContainer` |
| `--surface-1` | Фон панели превью (без бордера) | кастомный `colorFillTertiary` или отдельный токен |
| `--text-primary/secondary/muted` | Текст осн./вторич./приглушённый | `colorText`, `colorTextSecondary`, `colorTextTertiary` |
| `--border` / `--border-strong` / `--border-accent` | Границы обычные/усиленные/акцентные | `colorBorder`, `colorBorderSecondary`, `colorPrimary` |
| `--bg-accent` / `--fill-accent` / `--on-accent` | Фон акцентной плашки / заливка primary-кнопки / текст на ней | `colorPrimaryBg`, `colorPrimary`, `colorTextLightSolid` |

Значения, зафиксированные независимо от темы (не выводятся из Ant-дефолтов, задать явно):
- `border-radius` карточек — **12px**; `border-width` карточек — **0.5px**.
- Палитра свотчей (7 пресетов, фиксированные hex): `#f3c9d3` `#ffffff` `#1a1a1a` `#8a8a8a` `#7a1f1f` `#1f9e96` `#f2c200` + кастомный слот.
- Дефолтные цвета изделия: тело — `#1a1a1a`, отделка — `#7a1f1f`.
- Типографика: заголовки секций 13px/400; мелкие подписи (размеры S–7XL, подсказки формата файла) 11px; заметки-подсказки под блоками 12px; текст полей/чекбоксов 13px.
- Размер свотча — 24×24px, чекмарк-бейдж — 12×12px в правом нижнем углу.

## Components

| Component | Ant-компонент | Notes |
|---|---|---|
| Карточка-секция | `Card` (кастомная тема) | Дефолтный `Card` использует другой radius/border — нужен theme override, не дефолт |
| Тип изделия / Уход за материалом | `Radio.Group buttonStyle="solid"` | Нужны иконка+текст (тип) — кастомный рендер опций, не `optionType="button"` из коробки |
| Способ нанесения / Зона размещения | `Segmented` | Простой текст, дефолтного компонента достаточно |
| Цветовые свотчи | `ColorPicker` (presets) + кастомная обвязка | Ant `ColorPicker` по умолчанию — выпадающее меню, нужен инлайн-ряд кружков; кастомный слот «свой цвет» = один экземпляр `ColorPicker`. Чекмарк-бейдж и палитру — перенести как есть |
| Загрузка макета | `Upload.Dragger` | Обязательно проверить после интеграции: реальный файловый инпут должен остаться в таб-порядке (см. Accessibility Notes) |
| Таблица размеров | `InputNumber` × 10 в кастомной grid-обёртке | Нативные спиннеры скрыты в макете — проверить, нужно ли то же для `InputNumber` |
| Модалка «Таблица размеров» | `Modal` | Ничего из ручного JS (focus-trap/Escape) переносить не нужно — `Modal` даёт это из коробки |
| Кнопка отправки | `Button type="primary"` | Единственная filled-кнопка на экране |
| Кнопка экспорта | `Button` (default) | Secondary/outline |

## States and Interactions

| Element | State | Behavior |
|---|---|---|
| Кнопки-чипы (тип/способ/зона/уход) | Default / Selected | `border` → `border-accent`, `background` → `bg-accent`; для типа дополнительно цвет иконки |
| Кнопки-чипы | Keyboard | `role="radiogroup"`/`role="radio"`, roving tabindex, стрелки ←→/↑↓ переключают и сразу выбирают вариант |
| Свотч цвета | Default / Selected | `box-shadow` 1px→2px + чекмарк-бейдж появляется |
| Свотч «свой цвет» | Клик | Открывает нативный color picker; при выборе снимает выделение со всех пресетов в той же группе |
| Ткань (select) | `change`, значение содержит «2-х нитка» | Автоматически выбирает «Бережный уход» + `aria-live` объявление — не тихий сайд-эффект |
| Опции (капюшон/карман/молния/отделка) | Смена типа изделия | Скрываются/снимаются опции, недоступные для типа (маппинг ниже); сброс сопровождается синтетическим `change`, чтобы связанные SVG-элементы тоже спрятались |
| Опции: карман + молния вместе | Оба checked | Центральный карман на превью заменяется на два боковых |
| Кнопка «Отправить в сделку» | Всегда | Disabled-состояние сознательно не реализовано — гайдлайн хост-дизайн-системы против disabled-кнопок (низкий контраст, нет подсказки на тач). Реальная валидация при пустой форме — открытый вопрос для Этапа 2 |
| Модалка размеров | Открытие/закрытие | Фокус на кнопку закрытия при открытии, `Escape` закрывает, фокус возвращается на кнопку-триггер при закрытии |

### Маппинг доп.опций → тип изделия
- Футболка / Лонгслив → только «Отделка»
- Свитшот → «Молния» + «Отделка»
- Худи → все четыре («Капюшон», «Карман», «Молния», «Отделка»)

## Responsive Behavior

Не спроектировано ни для одного брейкпоинта кроме референсной ширины 680px. Действие для Этапа 2: замерить реальную ширину вкладки в Битрикс24 и явно решить — фиксированная минимальная ширина со скроллом, или брейкпоинт на одну колонку.

## Edge Cases (не реализовано, только текстовые формулировки на шаге ux-copy)

- **Файл больше 115 МБ** — валидация отсутствует в макете.
- **Неверный формат файла** — валидация только по `accept`-атрибуту, нужна проверка по содержимому (см. `specification.md`, раздел 4).
- **Нулевые размеры при отправке** — форма не проверяет, что указано хотя бы одно количество.
- **Недоступен API Битрикс24 при отправке** — не покрыто ни макетом, ни текстом ошибки.
- **Очень длинное имя файла** — есть `text-overflow: ellipsis`, не проверялось на реальных длинных именах.

Предложенные тексты ошибок (из `ux-copy`, реализовать в Этапе 1/2):
- Неверный формат: «Этот формат не поддерживается. Загрузите .png, .jpg, .pdf, .ai или .eps.»
- Файл больше лимита: «Файл больше 115 МБ. Сожмите файл или используйте более лёгкий формат.»
- Нет ни одного размера: «Укажите количество хотя бы для одного размера.»
- Нет макета (если станет обязательным): «Загрузите макет принта, чтобы отправить заказ.»
- Ошибка отправки: «Не удалось отправить в сделку. Проверьте соединение и попробуйте снова.»

| Element | Trigger | Animation | Duration |
|---|---|---|---|
| SVG-превью | Смена типа изделия (не худи) | `opacity` 1 → 0.35 | 150ms |
| Свотч (box-shadow) | Выбор/снятие | Мгновенный переход `box-shadow` | 100ms |

## Accessibility Notes

Полностью пройден WCAG 2.1 AA аудит (`design:accessibility-review`) — переносить эти паттерны 1:1, не переизобретать:
- Все поля/группы связаны с подписями через `label for` / `role="group|radiogroup" aria-labelledby`.
- Группы кнопок-чипов — `role="radiogroup"` + `role="radio"` + `aria-checked`, roving tabindex, навигация стрелками — при переходе на `Radio.Group`/`Segmented` это придёт из коробки автоматически, ручной JS не переносить.
- Модалка — `role="dialog" aria-modal aria-labelledby`, focus-trap, `Escape`, возврат фокуса — всё это даёт Ant `Modal` бесплатно.
- Зона загрузки файла — связана через `label for`, инпут visually-hidden (НЕ `display:none`), сохраняет таб-порядок. Это был реальный найденный баг (недоступная с клавиатуры зона) в ранней версии макета — проверить, что `Upload.Dragger` не воспроизводит его.

Полный список находок и фиксов аудита — в истории согласования (accessibility-review), не дублируется здесь.
